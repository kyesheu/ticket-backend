# v3.1 项目规格

状态：完成

## 目标

在 v3.0 RAG 基础上提供受控 AI 分诊：建议分类、优先级和处理人，由有权限用户确认后执行。

## 范围

- 返回分类、优先级、处理人建议、置信度、理由和来源。
- Java 提供合法分类和处理人候选集，Python 只能排序候选项。
- 用户可采纳、修改后采纳或拒绝建议。
- 采纳时重新校验权限、工单状态和候选有效性，并记录审计结果。

## 不做

- 自动分派、自动处理和自动关闭。
- Python 查询用户、部门、角色或工单数据库。
- 多步骤自主 Agent、模型工具调用和在线训练。

## 验收

1. 分类、优先级和处理人建议均来自服务端候选集。
2. 候选集外值、伪造来源和低置信度结果被拒绝或降级。
3. 用户确认前不修改工单；确认后复用现有 Service 完成变更。
4. 建议生成后工单或候选人变化时拒绝过期确认。
5. AI 不可用时原工单创建和分派流程正常。

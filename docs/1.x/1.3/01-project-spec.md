# v1.3 项目规格

状态：已完成

## 范围

- 复用 Ticket 五种角色数据范围：全部、自定义部门、本部门、本部门及下级、仅本人。
- 列表和按工单 ID 查询的接口统一执行对象级访问控制。
- 多个有效角色的数据范围取并集；本人创建或当前指派给本人的工单始终可见。

## 规则

- 使用工单创建时的 dept_id 快照，用户调岗不追溯历史工单。
- 仅启用且拥有相应 ticket 权限的角色可扩大范围。
- 服务端覆盖外部 dataScope，客户端不能指定用户、部门或 SQL。
- 无权访问和不存在统一返回“工单不存在”。

## 验收

五种范围、多角色并集、停用角色、参与人例外、跨部门分派、调岗快照和直接 ID 越权全部正确；v1.0–v1.2 回归通过。

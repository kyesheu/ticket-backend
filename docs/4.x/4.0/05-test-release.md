# v4.0 测试与发布

状态：规划中

## 测试目标

验证 AI 前置问答、转人工建单、自动分派和处理人闭环可以稳定运行，并且在 AI 服务不可用时不影响人工工单兜底。

## 功能测试

### AI 智能问答

- 输入常见问题，可以返回知识库答案。
- 输入历史工单相似问题，可以返回历史处理经验。
- 没有匹配知识时，提示需要转人工。
- AI 回答展示引用来源。
- AI 服务不可用时页面提示明确。

### 转人工建单

- 未解决问题可以转人工。
- 工单标题和内容自动生成。
- 原始问题、AI 回答和用户补充说明保留完整。
- 附件可以随转人工一起上传。
- 转人工后 AI 会话状态变为 `escalated`。

### 自动分派

- 分类明确且处理人存在时自动分派。
- 分类不明确时进入待分派。
- 处理人不存在或禁用时进入待分派。
- 置信度低于阈值时进入待分派。
- 自动分派结果写入操作日志。

### 处理人工作台

- 处理人可以看到自己的待办。
- 打开待办后可以看到 AI 辅助内容。
- 可以处理、回复、上传附件、转派和关闭。
- 关闭后工单状态正确更新。

### 知识沉淀

- 已关闭工单可以生成知识草稿。
- 知识审核后可以被 AI 问答检索。
- 禁用知识不会被 AI 引用。

## 集成测试

### 标准链路

```text
用户输入 WiFi 连不上
  -> AI 返回排查步骤
  -> 用户点击转人工
  -> 自动创建工单
  -> 自动分类为网络问题
  -> 自动分派给网络处理人
  -> 处理人查看 AI 建议
  -> 处理人填写处理结果
  -> 工单关闭
```

### 降级链路

```text
AI 服务不可用
  -> 用户点击转人工
  -> 创建普通工单
  -> 工单进入待分派
  -> 人工处理
```

### 低置信度链路

```text
用户输入模糊问题
  -> AI 不能确定分类
  -> 用户转人工
  -> 工单创建成功
  -> 不自动分派
  -> 管理员人工分派
```

## 非功能测试

- AI 问答接口超时控制。
- Python AI 服务失败重试和降级。
- 大文件知识库导入后的检索性能。
- 自动分派并发创建工单时的数据一致性。
- 日志中不出现密钥、token、完整敏感正文。
- 页面在常见分辨率下不出现内容溢出。

## 发布门禁

- 数据库迁移脚本可重复执行或具备明确版本控制。
- AI 服务不可用时普通工单流程可用。
- 自动分派可以通过配置关闭。
- 分派阈值可配置。
- 新增菜单权限脚本完整。
- 关键接口有基础测试覆盖。
- 前端普通用户、处理人、管理员入口清晰。

## 回滚策略

- 可以关闭 AI 问答入口，只保留原工单入口。
- 可以关闭自动分派，所有转人工工单进入待分派。
- 可以保留 AI 会话数据，不影响原工单表。
- Python AI 服务回滚不影响 Java 工单主流程。

## 上线后观察指标

- AI 问答次数。
- AI 直接解决率。
- 转人工率。
- 自动分派成功率。
- 自动分派改派率。
- 平均响应时间。
- 平均解决时间。
- 知识库引用次数。
- 用户重复提问率。

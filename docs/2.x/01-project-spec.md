# 01 — 项目规格

> 2.x | 2026-07-04 | 状态: ✅ 已完成

## v2.3 功能范围

| 功能 | 说明 |
|---|---|
| 全文检索 | 对工单编号、标题、描述和评论内容执行中文全文检索 |
| 条件过滤 | 支持分类、状态、优先级、创建人、指派人、部门和创建时间范围过滤 |
| 排序分页 | 默认按相关度和更新时间排序，也可按创建时间、更新时间排序；使用游标分页限制深分页 |
| 权限隔离 | 搜索结果与 MySQL 列表、详情使用相同对象级访问策略，ES 只负责召回候选工单 |
| 索引同步 | 工单创建、修改、流转、评论变更后通过事务事件表异步更新索引，支持失败重试 |
| 索引重建 | 管理员可触发全量重建并查询进度，使用版本化索引和别名原子切换 |
| 降级回源 | ES 不可用时明确返回检索服务暂不可用；普通结构化列表仍走 MySQL，不受影响 |

## v2.3 规则

- MySQL 是工单事实数据唯一来源，禁止将业务状态只写入 Elasticsearch。
- 业务事务只写 `ticket_search_event`，索引由异步任务构建，禁止在业务事务内双写 ES。
- 索引只保存检索所需快照，不保存附件内容、文件路径、操作日志或其他敏感字段。
- 搜索先按服务端生成的数据范围过滤 ES 候选，再通过 `ITicketAccessPolicy` 批量复核；接口不返回总命中数，无权数据不得通过高亮、游标或聚合泄露。
- 用户输入只能作为全文关键词和白名单过滤值，不接受 Query DSL、脚本、索引名或字段名。
- 单次页大小不超过 100；翻页使用 `search_after` 游标，不提供超出 Elasticsearch 结果窗口的页码跳转。
- 同一工单的索引事件按数据库事件 ID 幂等覆盖；旧事件不得覆盖新快照。
- 全量重建写入新版本索引，完成校验后切换固定别名；失败时保留当前线上索引。

## v2.3 不做什么

附件正文 OCR、向量检索、同义词后台管理、搜索建议、跨实体检索、统计聚合看板、RAG 和 Agent。

## v2.3 验收标准

1. 工单编号、标题、描述和评论关键词能召回对应工单，并返回安全高亮片段。
2. 关键词与分类、状态、优先级、用户、部门、时间过滤组合结果正确。
3. 当前用户只能看到与 MySQL 对象级访问规则一致的工单，越权数据不影响返回数量和翻页游标。
4. 创建、流转、修改和评论变更提交后最终同步到索引；事务回滚不产生索引更新。
5. 重复消费、乱序事件和进程重启不会生成重复文档或让旧数据覆盖新数据。
6. 同步失败按退避策略重试，超过上限进入失败状态并可由管理员补偿。
7. 全量重建期间搜索可用，成功后原子切换别名，失败时不影响当前索引。
8. ES 不可用不影响工单创建、处理和普通 MySQL 列表；检索接口返回明确可识别错误。

## v2.0 功能范围

| 功能 | 说明 |
|---|---|
| 流程模板 | 管理流程基本信息、节点、连线和发布版本 |
| 分类绑定 | 每个工单分类可绑定一个流程模板；未绑定时使用内置标准流程 |
| 版本快照 | 创建工单时锁定当时已发布版本，模板后续变更不影响存量实例 |
| 顺序节点 | 支持串行分派、处理和确认节点，不支持并行或会签 |
| 条件分支 | 支持按优先级、分类、创建人部门路由，并要求唯一默认分支 |
| 处理人规则 | 节点可指定用户、角色、创建人部门负责人、工单指派人或工单创建人 |
| 流程动作 | 支持完成当前节点、退回上一节点、取消和管理员终止 |
| 运行记录 | 保存流程实例、当前任务、处理人、处理意见和完整流转记录 |

## v2.0 规则

- 流程模板以 `workflow_key + version` 标识版本；草稿可编辑，发布后不可修改或删除。
- 同一流程模板同时只能有一个当前发布版本。发布新版本只影响之后创建的工单。
- 工单分类绑定稳定的 `workflow_key`，创建时解析当前发布版本并写入流程实例。
- 分类未绑定流程或绑定不可用时，使用系统内置标准流程；不得产生无流程工单。
- 条件仅读取服务端工单快照：`priority`、`category_id`、`dept_id`；不接受客户端 SQL 或表达式。
- 同一来源节点的条件按顺序匹配，最多命中一条；无条件命中时走唯一默认分支，否则事务回滚。
- 节点激活时解析处理人。用户、部门负责人、工单指派人和工单创建人固化为用户 ID；角色规则保留角色 ID，由具备该启用角色的用户处理。
- 业务状态继续使用 `NEW / PROCESSING / WAIT_CONFIRM / CLOSED / CANCELLED`：分派节点为 `NEW`，处理节点为 `PROCESSING`，确认节点为 `WAIT_CONFIRM`，到达结束节点为 `CLOSED`。
- 退回只允许退到当前实例已完成的直接上一人工节点；退回后重新生成任务，不回滚历史记录。
- 取消沿用创建人或管理员权限；终止仅管理员可执行，二者均结束流程并将工单置为 `CANCELLED`。
- v2.0 上线前的工单继续走原五状态流程，不创建流程实例，不迁移运行中数据。

## v2.0 不做什么

可视化拖拽设计器、并行网关、会签、或签、加签、转办、抄送、子流程、定时边界事件、任意脚本表达式、自定义字段条件、历史工单迁移、Flowable。

## v2.0 验收标准

1. 管理员可创建草稿、配置合法流程并发布；非法图、无默认分支或无处理人规则时拒绝发布。
2. 已发布版本不可修改；发布新版本后，旧工单仍按原版本继续运行。
3. 分类绑定流程后，新工单创建流程实例并生成唯一当前任务；未绑定分类使用标准流程。
4. 优先级、分类和创建人部门条件均能选择正确分支，且客户端不能注入条件表达式。
5. 五种处理人规则均正确鉴权，越权处理被拒绝。
6. 完成、退回、取消和终止在同一事务内更新任务、实例、工单状态及操作日志。
7. 重复提交同一任务不重复流转；并发处理时最多一个请求成功。
8. v1.0–v1.3 的 SLA、通知、评价、评论和数据权限回归通过。

## v2.1 功能范围

| 功能 | 说明 |
|---|---|
| 字段定义 | 管理分类下的自定义字段、排序、必填、默认值和启停状态 |
| 字段类型 | 支持文本、数字、日期、日期时间、单选、多选和布尔值 |
| 选项管理 | 单选和多选字段维护稳定选项值、展示标签、排序和启停状态 |
| 创建校验 | 创建工单时校验未知字段、重复字段、必填、类型、长度、数值范围和选项 |
| 值快照 | 保存字段 key、名称、类型、值及选项标签快照，配置变更不追溯历史工单 |
| 详情展示 | 工单详情按字段排序返回自定义字段值和历史快照元数据 |
| 流程条件 | 动态流程支持按 `CUSTOM_FIELD` 的字段 key 和值执行 `EQ / IN` 路由 |

## v2.1 规则

- 自定义字段归属一个工单分类；`field_key` 在分类内唯一，格式为大写字母、数字和下划线。
- 字段创建后 `field_key`、`field_type` 和所属分类不可修改；名称、校验参数、排序和状态可修改。
- 字段不物理删除，只允许停用。停用字段不再出现在新工单表单中，历史值继续展示。
- 创建工单只接受服务端为该分类返回的启用字段；未知 key、重复 key 和越权分类字段一律拒绝。
- 文本仅支持最大长度；数字支持最小值和最大值；v2.1 不接收用户正则表达式。
- 单选和多选值保存稳定 `option_value`，展示时同时保存 `option_label` 快照。
- 多选值规范化为有序 JSON 数组；同一选项不得重复。
- 工单创建、字段值保存和流程实例启动必须处于同一事务，任一步失败全部回滚。
- 流程自定义字段条件缺失或类型不兼容时视为不命中，最终走唯一默认分支。

## v2.1 不做什么

字段公式、级联字段、动态正则、文件字段、富文本、字段级数据权限、列表动态列、按自定义字段筛选、批量导入、历史值回填。

## v2.1 验收标准

1. 管理员可为分类维护七种字段及选项，非法 key、重复 key 和非法校验参数被拒绝。
2. 创建工单时必填、类型、长度、数值范围、未知字段、重复字段和无效选项校验正确。
3. 工单、字段值和流程实例同事务保存，失败时不产生半成品数据。
4. 字段定义或选项修改后，历史工单仍按创建时快照展示。
5. 禁用字段不影响历史详情，且新工单不能提交该字段。
6. `CUSTOM_FIELD` 条件可正确执行文本、数字、选项和布尔值的 `EQ / IN` 路由。
7. 客户端不能通过字段 key、值或条件注入 SQL/脚本。
8. v1.0–v2.0 单元测试和完整 smoke 全部回归通过。

## v2.2 功能范围与规则

- 工单正文和评论均可关联附件；附件元数据保存在数据库，文件内容复用 RuoYi 本地文件存储。
- 保存原始文件名、相对存储路径、MIME、扩展名、大小和上传人快照，不向客户端暴露绝对路径。
- 临时上传按登录用户记录归属；列表、下载、绑定和删除均先执行工单对象级访问控制。
- 客户端不能直接提交服务器路径；临时附件只能由上传人绑定一次，且不得跨工单或跨业务类型绑定。
- 附件采用逻辑删除；终态工单附件默认不可删除，管理员除外。
- v2.2 不做分片上传、断点续传、秒传、在线预览、病毒扫描、对象存储或历史附件迁移。

## v2.2 验收标准

1. 有权用户可上传附件并绑定到工单正文或评论，非法类型、空文件和超限文件被拒绝。
2. 列表和下载遵守工单对象级权限，且下载响应不泄露服务器绝对路径。
3. 临时附件不能被他人、重复或跨工单绑定，业务保存失败时不产生错误绑定。
4. 原始文件名和文件元数据在绑定后保持快照，存储文件名不可由客户端控制。
5. 删除权限、终态限制和重复删除语义正确，物理文件清理失败可追踪。
6. v1.0–v2.1 单元测试和完整 smoke 全部回归通过。

# v3.2 项目规格

状态：完成

## 目标

让知识库和 AI 建议可管理、可反馈、可评测，形成运营闭环。

## 范围

- 管理知识文档列表、详情、状态、重新导入和删除。
- 展示文档切片数、内容摘要、最近导入结果和失败原因摘要。
- 用户对处理建议、回复草稿和分诊建议提交有用性反馈。
- 统计建议生成量、降级率、采纳率、有用率和主要失败原因。
- 固定评测集支持版本化执行和结果对比。

## 不做

- 在线训练、自动微调和模型自动切换。
- 复杂知识审批发布流、段落级 ACL 和多人标注平台。
- 将模型调用成本或运营指标作为工单业务事实。

## 验收

知识文档生命周期、失败恢复、删除后不可检索、反馈幂等、权限隔离、指标口径和评测回归正确；运营能力故障不影响工单主流程。

## 阶段 54 范围

- 固化 Java 与 Python 的知识文档管理契约：列表、详情、导入结果摘要字段。
- 新增反馈表 SQL、Java 模型、Mapper 和枚举骨架。
- 新增契约与 SQL 结构测试，先覆盖权限入口、字段约束、唯一键和参数边界。
- 不实现删除、重导、失败恢复、反馈提交和运营指标聚合。

# 01 — 项目规格

> 3.x | 2026-07-04 | 状态: 📝 规划中

## v3.0 功能范围

| 功能 | 说明 |
|---|---|
| 知识库文档导入 | 管理员上传文档，由 Python 服务解析、切片、向量化并写入向量库 |
| 相似知识检索 | 根据工单标题和描述检索相关知识文档及相似历史工单 |
| AI 处理建议 | 基于当前工单和检索结果生成供处理人参考的排查、处置建议 |
| AI 回复草稿 | 基于同一证据生成可编辑的工单回复草稿，由处理人确认后手工提交 |

## v3.0 规则

- Java 工单系统负责登录鉴权、工单权限、业务状态、上下文组装和结果展示，不实现切片、向量化、检索编排或提示词逻辑。
- 独立 Python FastAPI 服务负责 LangChain、文档解析、向量化、相似检索和大模型生成；Java 通过内部 HTTP 接口调用。
- Java 只向 Python 传递当前用户已获权的最小工单上下文；Python 不直接连接工单业务表，不自行判断工单权限。
- 历史工单只有在 `CLOSED` 且存在有效处理结果时才可进入相似工单知识源；同步内容不包含附件路径、内部权限信息或无关个人数据。
- 处理建议和回复草稿必须返回知识来源；检索无有效证据、Python 服务超时或模型不可用时返回明确降级结果。
- AI 输出只作为草稿展示，不自动分派、处理、评论、确认、取消或关闭工单。
- Java 与 Python 使用版本化内部 HTTP 契约，配置连接超时、读取超时和响应大小上限；AI 失败不得影响工单主流程。
- 模型、Embedding 和向量库凭据仅存在 Python 服务外部配置中，不进入 Java 配置、数据库或日志。

## v3.0 不做什么

复杂 Agent、自动处理工单、自动状态流转、自动评论、模型训练/微调、知识图谱、LangGraph 工作流、多模型路由、段落级权限、多轮会话、反馈系统、OCR、互联网搜索和任意代码执行。

## v3.0 验收标准

1. 管理员可导入受支持文档，Python 服务完成解析、切片、向量化和入库；非法、空白或超限文档被拒绝。
2. 输入工单问题后，能召回相关知识文档和相似已关闭工单，结果包含可定位来源。
3. AI 能基于当前工单和检索证据生成处理建议，证据不足时明确说明，不伪造依据。
4. AI 能生成回复草稿，但不会自动写评论或触发任何工单状态流转。
5. Java 不包含 LangChain、模型 SDK、向量检索和提示词实现；Python 不直接修改工单业务数据。
6. 越权工单不能调用辅助接口；发送给 Python 的上下文仅包含当前用户可访问的数据。
7. Python、向量库或大模型不可用时，Java 返回明确降级信息，既有工单业务继续可用。
8. Java 全量测试与 smoke、Python 单元/接口测试及 Java→Python 端到端 smoke 全部通过。
